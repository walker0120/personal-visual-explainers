# Diagram Yakuzaishi

薬の情報を**薬剤師が患者さんに説明できるレベル**まで整理したHTMLを生成する。
新人からベテランまで、誰でも同じクオリティで服薬指導ができることがゴール。

## 依頼の典型パターン

- 「〇〇（薬名）と△△（薬名）の違いを図解して」
- 「〇〇の副作用を図解して」
- 「〇〇から△△に薬が変わった患者さんへの説明図解を作って」
- 「〇〇の服薬指導用図解を作って」

---

## モード判定（最初に確認）

依頼を受けたら、まず以下の2モードを判断して実行する。

| モード | 条件 | 出力 |
|-------|------|------|
| **クイックモード**（デフォルト） | 通常の依頼 | クイックカード  |
| **フルレビューモード** | 「詳しくレビューして」「しっかり確認して」など明示 | フル版 → レビュー → ブラッシュアップ |

**レビュー（Phase 7-8）は明示的に求められたときのみ実行する。**

---

## ワークフロー

### Phase 0: リサーチ＋HTML骨格を並列起動（案②）

**exploreサブエージェント**と**HTML骨格の準備**を同時に始める。
サブエージェントの結果を待つ間に、骨格（ヘッダー・セクション構造）を先に組む。

```
// 並列実行
Task({
  subagent_type: "explore",
  description: "薬剤情報の調査",
  prompt: `
    以下の薬剤について調査してください：
    【対象】{薬名 or URL}

    調査項目：
    1. この薬は何のための薬か（適応症・目的）
    2. 主な作用機序（患者向けに言い換えると何をする薬か）
    3. よくある副作用TOP5（頻度・程度・対処法）
    4. 服薬上の注意点（食事・時間・禁忌など）
    5. 比較薬がある場合：違い・使い分けのポイント
    6. 患者さんがよく心配すること・よくある質問

    結果は箇条書きで整理して返してください。
  `
})

// 同時に：HTML骨格（ヘッダー・セクション枠）の準備を開始
```

### Phase 1: 情報整理

リサーチ結果をもとに：
1. 情報の全体像を把握
2. 患者さんの視点で「わかりにくそうな概念」を洗い出す
3. 比較が必要な薬がある場合は共通点・相違点を整理
4. **「患者さんに必ず伝えること3つ」と「比較表の項目」を先に確定させる**（クイックカードに使う）

---

### ★ Phase Q: クイックカード出力（案①・第1弾）

Phase 1が終わったら、**フル版を待たずに即座に**クイックカードを出力する。

**クイックカードに含む内容：**
1. 30秒まとめ（3点）
2. 患者さんに必ず伝えること（3項目）
3. 比較表（ある場合）
4. 受診タイミング（赤ボックス）

**出力先：**
```
output/{薬名}/quick-card.html
```

ファイルを保存したら「**クイックカードを保存しました。続けてフル版を作成中です…**」とユーザーに伝える。

クイックカードのHTML構造 → [references/quick-card-template.md](references/quick-card-template.md)

---

### Phase 2: 用語翻訳表を作成

医療用語を患者さんにも伝わる言葉に変換する。

| 医療用語 | やさしい言葉 | たとえ |
|---------|------------|--------|
| 血圧降下薬 | 血圧を下げる薬 | 水道管の水圧を調整する弁 |
| 副作用 | 薬の意図しない働き | 車の排気ガスみたいなもの |
| 半減期 | 薬の効果が半分になる時間 | 氷が溶けていくイメージ |

**詳細** → [references/term-dictionary.md](references/term-dictionary.md)

### Phase 3: たとえ話を3つ以上考える

| 種類 | 例 |
|-----|-----|
| 身近な例 | 家電・料理・日常の道具 |
| 身体で起きることを視覚化 | 体の中でどう動くか |
| 患者さんが安心できる例 | 「怖くない」という方向の例え |

### Phase 4: 服薬指導のポイントを絞り込む

- 「患者さんに必ず伝えること3つ」を確定（Phase 1で仮決めしたものを精査）
- 「飲み忘れ・飲み合わせ・生活上の注意」などの実務ポイントを優先
- 副作用は「どんな時に受診すべきか」まで含める

### Phase 5: キャラクター対話を設計

| キャラクター | 役割 | 表情の使い分け |
|------------|------|--------------|
| ルーキーさん | 新人薬剤師。経験浅く、患者さんへの説明に迷うことがある | 疑問、驚き、理解、不安 |
| センパイさん | ベテラン薬剤師。わかりやすい比喩で本質を伝える | 説明、諭す、確認 |

キャラクター画像は `./images/` に配置されている場合は使用する。
ない場合はイニシャルアバター（R / S）で代替する。

**詳細** → [references/character-usage.md](references/character-usage.md)

### Phase 6: フル版HTML生成

1. Tailwind CSS CDN使用
2. **Lucide icon**を使用（絵文字禁止）
3. 医療・薬局ブランドカラー（ティールグリーン系）適用
4. 用語解説ボックス配置
5. 比較表・フローチャートを積極的に使う
6. クイックカードの内容を冒頭に含め、その後にキャラ対話・詳細セクションを続ける

**出力先：**
```
output/{薬名}/index.html
```

**詳細** → [references/html-structure.md](references/html-structure.md)

---

### Phase 7: 批判的レビュー【任意・案③】

**「詳しくレビューして」「しっかり確認して」と言われたときのみ実行する。**
通常のクイックモードではスキップする。

```
Task({
  subagent_type: "generalPurpose",
  description: "服薬指導図解の批判的レビュー",
  readonly: true,
  prompt: `
    あなたは「薬について詳しくない60代の患者さん」と「入局1ヶ月の新人薬剤師」の2人の視点でレビューします。

    以下のHTML図解を読んで、忖度なく批判してください：

    【HTMLファイルパス】{path}

    ## レビュー観点

    患者さん視点：
    1. わからない言葉：薬の専門用語が残っていないか
    2. 不安になる表現：副作用の書き方が怖すぎないか
    3. 行動がわからない：「じゃあ私はどうすれば？」がわかるか

    新人薬剤師視点：
    4. 説明の順番：患者さんに話す順番として自然か
    5. 抜け漏れ：服薬指導として必要な情報が揃っているか
    6. 使いやすさ：図解を見ながら説明できるか

    ## 出力形式

    ### 致命的な問題（これがあると使えない）
    - ...

    ### 改善すべき点（直せばもっと良くなる）
    - ...

    ### 良かった点（このまま残すべき）
    - ...

    ### 総評
    5段階評価: ★★★☆☆（3/5）のように
    一言コメント
  `
})
```

### Phase 8: ブラッシュアップ【任意・Phase 7を実行した場合のみ】

Phase 7のレビュー結果を踏まえてHTMLを修正する。

**優先度**:
1. 致命的な問題 → 必ず修正
2. 改善すべき点 → 可能な限り対応
3. 良かった点 → 維持

## 品質チェックリスト

### クイックカード（必須）

- [ ] 30秒まとめ（3点）がある
- [ ] 「患者さんに必ず伝えること3つ」がある
- [ ] 比較表がある（比較が必要な薬の場合）
- [ ] 受診タイミングの赤ボックスがある

### フル版（必須）

- [ ] 医療用語には必ず「用語解説ボックス」がある
- [ ] 服薬上の注意（食事・時間・禁忌）が含まれている
- [ ] キャラクター対話が導入・中間・まとめにある
- [ ] スマホでも読みやすいレスポンシブ対応

### デザイン（両方共通）

- [ ] Lucide iconを使用（絵文字禁止）
- [ ] 医療・薬局ブランドカラーが適用されている
