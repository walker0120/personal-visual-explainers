# Diagram 薬局

薬の情報（比較・副作用・用法）を**患者様でも直感的に理解できるHTML**に変換する。
テンプレートへのデータ流し込みで完結するため、出力まで1分以内を目指す。

## 依存

- `references/base.html` — クイックカードHTMLテンプレート（構造・スタイル完成済み）
- `references/term-dictionary.md` — 医学用語→患者向け言い換え辞書

---

## ワークフロー

### Phase 0: テンプレートを読む

`references/base.html` を読み、プレースホルダー（`{{...}}`）の一覧を把握する。
HTMLの構造は変更しない。テキスト内の `{{...}}` を薬のデータで置き換えるだけ。

### Phase 1: 薬の情報を集める

**薬剤師が情報を持っている場合** → そのまま Phase 2 へ。検索不要。

**情報が不足している場合のみ**、Web検索を1回行う:

```
"{薬名A} {薬名B} 違い 使い分け 副作用"
```

収集する情報（必要最小限）:
- 薬の表示名・一般名
- 主な適応症の違い（比較表の行になるもの5〜8項目）
- 各薬の患者指導3点（最も重要な順）
- 両薬に共通する緊急症状6つ

### Phase 2: プレースホルダーを埋める

`references/base.html` の内容をコピーし、以下の `{{...}}` を全て置き換える。

**ヘッダー:**
| プレースホルダー | 内容 |
|----------------|------|
| `{{薬A}}` / `{{薬B}}` | 短縮表示名（例: レキサルティ） |
| `{{サブタイトル}}` | 薬効カテゴリ＋目的 1行（例: 非定型抗精神病薬の使い分けと患者指導ポイント） |

**30秒まとめ（3項目）:**
| プレースホルダー | 内容 |
|----------------|------|
| `{{まとめN題}}` | 5〜10文字の見出し |
| `{{まとめN文}}` | 20〜40文字の本文 |

**薬A・薬B 患者指導（各3項目）:**
| プレースホルダー | 内容 |
|----------------|------|
| `{{薬A名}}` / `{{薬A一般名}}` | 表示名・一般名（例: エビリファイ等 / アリピプラゾール） |
| `{{薬A指導Nタイトル}}` | 患者への一言（20文字以内） |
| `{{薬A指導N説明}}` | 理由・対処法（50文字以内） |

患者指導の順番は固定:
- 1番（赤/必ず）: 最も重要な副作用・注意点
- 2番（青/重要）: 効果・服用方法
- 3番（橙/注意）: 生活上の注意

**比較表:**

`TABLE_ROWS_START` 〜 `TABLE_ROWS_END` のコメント間にある `<tr>` 行を薬のデータで置き換える。
行数は5〜10行が目安。`bg-teal-50` が薬A列、`bg-purple-50` が薬B列。

値のスタイルガイド:
- 適応あり → `class="text-green-700 font-medium"` ＋ `✓ 適応あり`
- 適応なし → `class="text-gray-400"` ＋ `－ 適応なし`
- 注意が必要 → `class="text-orange-600 font-medium"`
- 特に問題なし → `class="text-gray-600"`

**緊急症状（6項目）:**

`{{緊急症状1}}` 〜 `{{緊急症状6}}` を置き換える。
`<strong>症状名</strong>（括弧で補足）` の形式で書く。

**専門用語が出たら:** `references/term-dictionary.md` を参照して患者向けに言い換える。

### Phase 3: ファイル保存

1. `output/` ディレクトリがなければ作成する
2. スラッグを決める（例: `rexulti-vs-abilify`, `metformin-caution`）
3. `output/{スラッグ}/quick-card.html` として保存する

フル版（index.html）を作らない場合は、テンプレート末尾の「フル版リンク」ブロックを削除する。

### Phase 4: デプロイ

Node.js の有無を確認:

```bash
node --version
```

**Windows（PowerShell）:**

```powershell
npx --yes surge output/{スラッグ} --domain diagram-{スラッグ}.surge.sh
```

**macOS / Git Bash:**

```bash
npx --yes surge output/{スラッグ} --domain diagram-{スラッグ}.surge.sh
```

初回のみ、ターミナルにメールアドレスとパスワードの入力を求められる。

### Phase 5: 完了報告

```
完成・公開完了: 【薬名A】vs【薬名B】 服薬指導クイックカード

公開URL:
https://diagram-{スラッグ}.surge.sh/quick-card.html

患者さんへの説明ポイント:
- ...
```

## 品質チェックリスト

- [ ] `{{...}}` プレースホルダーが全て置き換えられている（残っていないか）
- [ ] 30秒まとめが3項目ある
- [ ] 薬A・薬Bそれぞれ3項目の患者指導がある
- [ ] 比較表が5行以上ある
- [ ] 緊急症状が6項目ある
- [ ] 専門用語に患者向けの言い換えがある
- [ ] フル版がない場合、フル版リンクブロックを削除している
